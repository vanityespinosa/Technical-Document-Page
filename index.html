Introduction

Git is a type of version control system (VCS) that tracks changes in computer files and is useful for coordinating work between multiple users. It is hardly the first VCS out there but is by far the most popular one.

It can track what changed in an edit, who made that change, and why they made that change. From writing a novel, to creating code, to editing some design files, Git can track changes in all of these instance, and allow others to see and add to these changes. It is not just for programming; it can be used to make different versions of the work you do on a computer.

How Git Works

When Git is intialized in a project, it creates a repository and stores that project's current state within it.

After a user makes changes to a project in a repository, Git tracks those changes. To add these changes to the repository, a user must commit those changes to the repository. This creates a "checkpoint" or a branch that the user or someone else can branch off of (get it?).

Once all changes have been complete, branches that are deemed necessary or correct can be merged into the master branch. After that, a user can push these changes to the repository or pull changes for the repository.

Hosting Services

Some public Git hosting services include Bitbucket, GitHub, and Gitlab. These kind of hosting services are called repository management services.

These public hosting services are a great way to not only show one's work but to collaborate and connect with other developers in a public place. This is how a lot of open-source projects are coordinated; it is easy for users to branch from a project they find interesting and start putting their own stamp on an open-source project.

Here are some simple rules to always following when using one of these public repositories:

  • Rule 1: Create a Git repository for every new project.
  • Rule 2: Create a new branch for every new feature.
Setting Up Git

The good thing about Git is that it comes as a default to many systems. However, if you don't already have it:

  • GitHub has a nice graphical user interface (GUI) called GitHub Desktop. It is simpler to use but you won't be able to see what is really going on.
  • Otherwise, Git's command-line interface (CLI), though intimidating at first, is the recommended option for beginner and advanced users. It allows you to get familiar the terminal no matter what device you are using.

Common Commands

There are some basic commands to get you started:

Initializing a new repository

git init

Adding a file or files

  • git add index.html
  • git add .

Committing a change

git commit -m "Your message."

Getting a status

git status

Checking out commits or branches

git log
